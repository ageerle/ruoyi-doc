# 编排管理

::: info 你正在阅读的是 [RuoYi AI](https://gitee.com/ageerle/ruoyi-ai) 的文档！

- 如发现文档有误，欢迎提交 [issue](https://gitee.com/ageerle/ruoyi-ai/issues) 帮助我们改进。

## 概览

编排管理用于把模型、知识库、工具、人工反馈和外部系统调用组织成固定流程。RuoYi AI 的 `ruoyi-aiflow` 模块基于 LangGraph4j 构建工作流引擎，支持可视化节点、条件分支和流式执行。

```text
Workflow
  -> Node
  -> Edge
  -> Runtime
  -> SSE 输出
```

## 固定流程管理

固定流程适合业务步骤明确、可复用、需要审计的场景，例如：

- 文档上传后自动解析、摘要、入库。
- 客服问题先分类，再检索知识库，最后生成回复。
- 数据查询先生成 SQL，再执行查询，最后生成图表。
- 内容生成先规划，再生成图片/视频，再人工确认。

## 节点类型

当前代码中已包含以下节点：

| 节点 | 说明 |
| --- | --- |
| `StartNode` / `EndNode` | 工作流入口与出口。 |
| `LLMAnswerNode` | 大模型回答节点。 |
| `ImageNode` | 图像生成节点。 |
| `KnowledgeRetrievalNode` | 知识库检索节点。 |
| `KeywordExtractorNode` | 关键词抽取节点。 |
| `ClassifierNode` | 内容分类节点。 |
| `SwitcherNode` | 条件分支节点。 |
| `HumanFeedbackNode` | 人工反馈节点。 |
| `HttpRequestNode` | HTTP 请求节点。 |
| `MailSendNode` | 邮件发送节点。 |

## 数据流

节点之间通过输入输出数据传递。建议在 UI 中显式展示：

| 项目 | 说明 |
| --- | --- |
| 输入变量 | 当前节点需要的参数。 |
| 输出变量 | 当前节点执行后产出的字段。 |
| 引用表达式 | 引用上游节点输出。 |
| 类型 | 文本、数字、布尔、文件、选项。 |

## 执行状态

工作流执行建议保留以下状态：

| 状态 | 说明 |
| --- | --- |
| 等待 | 节点尚未满足执行条件。 |
| 运行中 | 节点正在执行。 |
| 成功 | 节点完成并产出结果。 |
| 失败 | 节点异常，需要错误信息。 |
| 中断 | 等待人工反馈或外部回调。 |

## 与智能体的区别

| 对比项 | 工作流编排 | 智能体 |
| --- | --- | --- |
| 控制方式 | 人预先定义流程。 | 模型动态规划步骤。 |
| 稳定性 | 更可控，适合生产流程。 | 更灵活，适合开放任务。 |
| 审计 | 节点和边天然可审计。 | 需要记录推理、工具调用和观察结果。 |
| 适用场景 | 审批、生成流水线、固定业务链路。 | 数据分析、搜索、复杂问答、多工具协作。 |
